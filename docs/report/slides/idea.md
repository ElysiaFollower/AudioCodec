## 第一节： 先讲assumption 和 introduction
就是报告中提过的——speech是一个特定子分布，所以我们认为neural codec理应能训出更好的东西。这一部分是很有insight值得讲的、

所以我们这个项目就是使用神经网络来做一个 encoder+decoder


## 第二节： 主要讲实现思路和技术细节

不用精确到每个参数怎么选择， 但是要展示清楚主干网。

先一个小节展示一个最核心的架构， 即我们最开始的架构， 自编码器 + RVQ + 解码器

在这个架构上我们可以讲清楚训练为什么能够进行的核心 —— 确保没接触过的人也能懂个大概。
我会在这里说， 这个思想和VAE很像，  我们会把原始音频切割成一个个帧向量， 然后将其输入到encoder中， 然后就到了中间的bottleneck(这里我会对着架构图讲， 所以架构图最好是中间窄的)， 于是帧向量变成了更短的向量， 就实现了压缩，即编码的过程，再然后我们把压缩后的向量送给decoder， 它会基于这些token重建出目标音频， 这就是解码的过程。 
核心思想是这样。
然后我们就可以再仔细看一下bottleneck这一层在发生什么。最简单的方法当然只是压缩到一个更短的隐向量， 但是这里我们使用了RVQ残差向量量化的技术。构建可自定义数量的码本， 然后使用这些码本的码字逐级对压缩后的向量进行编码。
具体来说就是 x 量化成 r1, r2, r3, r4。。。, 最后重建的时候就是将它们加起来。
x-r1作为下一层的输入得到r2， 然后x-r1-r2作为下一层的输入得到r3， 以此类推。

然后新开一节展示完整架构
主要描述相较于基本盘backbone变更了什么——SEANet、更高级的损失计算啊这些。
承认在原来的基本盘上能够重建但质量不佳。 经过这些优化细节之后，才真正实现了高保真的重建

## 第三节：主要讲实验结果

把报告中benchmark的全量表格放出来， 直观说明neural codec实现了传统算法无可匹敌的压缩率， 并在相似码率下实现了更高的重建质量 —— xx评测指标更好。 定性分析一下就行。

折线图也可以放一下。 黑线是neural codec，只有它能实现真正的超低码率， 并且在数值指标上保持了极高的优势， 只有部分传统算法在远大于它的码率下才能在指标上取得微弱的提升。
但就人耳试听结果来看， 即便是最低码率的neural-2k都能在大多数情况下让我真假难辨甚至主观听感要由于其他码率更高但同样低码率的传统算法的结果。

## 第四节： 主要讲局限性

就是neural-2k的声线漂移现象。
直接把那个图贴上来， 然后分析一下。

前两列分别是原始音频的频谱，最后一列是什么diff(这个你得跟我解释一下，因为我没看懂)。
前两行是从样本中特定找出来的声线偏移的时间段，从图像中可以看到直接看频谱分布基本是相似的，没有明显的区别， 说明训练还是基本可靠的，事实上他也的确能够还原出语言信息， 只是声线偏了。 然后通过log-spectral diff进行更深层次的分析， 可以发现的确这两份重建样本和原样本存在某种意义上的误差， 而这个误差在最后一行的成功重建样本上是无法明显观察到的。

这是我们项目的局限性。 但同时也意味着，既然这个误差是可以通过这样的方法直观衡量出来的， 也就是说他是可以作为loss升格为优化目标的。如果从这里入手， 或许可以解决这个问题。但受限于时间原因，这里不深入进行探讨。
