# 项目总览

## 1. 项目定位

- 当前课程目标：在 1 到 2 天内完成一个可训练、可评测、可展示的 `speech neural codec` baseline，核心主干是 `Encoder-Decoder + RVQ`，并与 `MP3` 做对比实验。

## 2. 核心目标

本次课程项目的核心目标不是追求论文级创新，而是完成一个“能跑通、能解释、能对比”的最小闭环：

1. 训练一个仅面向语音的神经编解码器。
2. 展示从波形到离散 token 再到重建音频的完整流程。
3. 用客观指标和听感样例，对比神经 codec 与传统 `MP3` 的差异。
4. 总结 learned compression 与传统压缩方法在设计哲学上的差别。

## 3. 范围

### In scope

- 单声道 `16 kHz` speech。
- 小规模数据集或数据子集上的快速训练与验证。
- `Encoder-Decoder + RVQ` 的基础神经 codec。
- 与 `MP3` 的压缩率、重建质量、可视化对比。
- 一项低风险扩展或消融分析。

### Out of scope

- 通用音频场景（音乐、环境声、多说话人混合）。
- 流式解码、低延迟部署、实时系统优化。
- 追求 SOTA 结果或大规模预训练。
- 在本次课程周期内直接实现 Mamba 主干。

## 4. 交付物

建议将最终交付控制在下面五类：

1. 训练代码：数据加载、模型、损失、训练循环。
2. 评测代码：压缩率、重建质量、音频样例导出。
3. 实验结果：表格、频谱图、若干听感样例。
4. Presentation：问题背景、方法、实验、结论、未来工作。
5. 项目文档：范围、架构、风险、计划。

## 5. 成功标准

满足以下条件即可视为本次课程目标完成：

1. 模型能在小规模 speech 数据上稳定收敛，不出现明显训练崩坏。
2. 能导出离散 token，并完成从 token 到重建音频的解码。
3. 能给出至少一组和 `MP3` 的公平对比。
4. 能清楚解释模型结构、压缩路径、评价指标和实验结论。

## 6. 推荐实验故事线

建议 Presentation 采用下面的故事线：

1. 传统音频压缩依赖手工设计的变换、量化和熵编码。
2. 神经 codec 通过学习式表示，将“特征提取 + 压缩 + 重建”统一到端到端训练中。
3. `RVQ` 提供离散 token 序列，天然适合后续与生成模型或 LLM 结合。
4. 在课程作业约束下，先聚焦 speech 场景，做一个小而完整的 neural codec baseline。
5. 对比 `MP3` 后，总结 learned codec 的优点、局限和下一步改进方向。

## 7. 推荐的低风险“创新点”

不要把“创新”理解成一定要发明新模型。对课程项目来说，更稳妥的是在 baseline 之上做一项清晰的分析：

- 分析 `RVQ stage` 数量对码率与质量的影响。
- 对比 `L1 waveform loss` 和 `multi-scale STFT loss`。
- 对比单级量化和残差量化。

这类工作实现成本低，但足以展示你对神经压缩设计空间的理解。

## 8. 当前冻结的默认实验设置

为了减少实现阶段的歧义，当前课程项目默认按下面这组配置执行：

- 数据集：`LibriSpeech dev-clean`
- 切分：`train 60 min / val 10 min / test 10 min`
- 模型：`1D Conv Encoder -> RVQ -> 1D Conv Decoder`
- 关键参数：`latent_dim=128`、`codebook_size=256`、`num_quantizers=8`
- 训练预算：`500-step smoke test + 5000-step main run`
- 扩展预算：若验证集仍改善，再继续到 `10000 steps`

除非出现明确阻塞，不建议在实现过程中临时更换数据集或模型主干。
